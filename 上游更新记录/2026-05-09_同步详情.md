## 🎯 对本项目影响评估

### 达妙路径

✅ **完全不受影响**——所有改动都在 `add_robstride_motor` / `robstride_*` / `motor_vendors/robstride/`。`add_damiao_motor` / `Mode` / `Controller` / `Motor` 核心 API 未动。

### 配置兼容性

`reBotArm_control_py/config/arm.yaml` 和 `gripper.yaml`：

- 全部 `vendor: "damiao"`，不走 RobStride 分支
- 即便将来切到 RobStride：`motor_id` 0x01..0x07（1..7）、`feedback_id` 0x11..0x17（17..23）都在新版校验范围内（1..255 / 0..255）

### Python 环境

- 升级前：Windows Python 3.12 + WSL2 Ubuntu Python 3.10 都是 **wheel 0.2.4**
- 升级方式（**已确认无需 Rust 工具链**）：上游 motorbridge 在 PyPI 已发布 0.2.6 预编译 wheel
  ```bash
  # Windows
  pip install --upgrade motorbridge

  # WSL2
  pip3 install --upgrade motorbridge
  ```
- 升级后：Windows + WSL2 两端均为 **wheel 0.2.6**，submodule 源码层与 Python 环境层一致

> **避坑提示**：之前 5-05 升级 0.1.7→0.2.4 用 `pip install -e software/MotorBridge/bindings/python` 走的是**本地 editable 模式**，这条路径要求本地有预编译的 motor_abi 库（cargo build 产物）。但项目作者也在 PyPI 发预编译 wheel，直接 `pip install --upgrade motorbridge` 是更简单的路径，本机无需任何 Rust 工具链。两种装法的结果模块都在 `site-packages/motorbridge/`，差异在于 wheel 模式直接拷贝预编译 .dll/.so。
