### 3. 修正：安装 motorbridge Python 包到 v0.2.4

**背景**：4-29 升级 `software/MotorBridge` submodule 到 v0.2.4，但**只升级了源码**，Python 环境里还是 4-13 装的 v0.1.7 旧版。今天发现并补装。

**之前给错的命令**：`pip install -e software/MotorBridge`
**正确路径**：`pip install -e software/MotorBridge/bindings/python`（Python SDK 在 bindings/ 子目录里，主目录是 Rust 项目）

**WSL2 安装结果**：
```
Found existing installation: motorbridge 0.1.7
Successfully installed motorbridge-0.2.4
```

**验证通过**：
- `pip show motorbridge` → Version: 0.2.4 ✅
- `from motorbridge import Controller, Mode; print('OK')` → OK ✅
- `dir(motorbridge)` 列出完整 API：Controller / Motor / Mode / MotorState + 全套 RID_* 常量 + 达妙专用工具

**新版的好东西**（旧版没有）：
- 寄存器命名常量：`RID_KP_ASR=25`、`RID_KI_ASR=26`、`RID_KP_APR=27`、`RID_KI_APR=28`、`RID_PMAX/VMAX/TMAX`、`RID_CTRL_MODE`、`RID_ESC_ID`、`RID_MST_ID`、`RID_TIMEOUT`
- 达妙专用工具：`DAMIAO_HIGH_IMPACT_RIDS`（高风险写错会失控的）/ `DAMIAO_PROTECTION_RIDS`（保护类）/ `DAMIAO_RW_REGISTERS`（全部可读写）/ `get_damiao_register_spec()`

**新版的小坑**：
- 不再提供 `motorbridge.__version__`，查版本必须用 `pip show motorbridge`

**Why**：`pip install -e software/MotorBridge` 会失败因为根目录是 Rust 项目（`Cargo.toml`），Python SDK 在 `bindings/python/`。

### 2. 调试达妙电机 DM-J4310-2EC，发现调试工具版本问题

**问题现象**：
- 使用 `Debugging_Tools_v.1.6.8.8.exe` 调试电机时，串口在校准等操作中频繁出现乱码

- **调试工具版本不兼容**！`Debugging_Tools_v.1.6.8.8.exe` 与电机固件不匹配
- 正确工具应使用：`DMTool v2.1.6.0-FD加速(1).exe`

**工具位置**：
- 错误版本：`hardware/测试工具/Debugging_Tools_v.1.6.8.8.exe`（不要用）
- ✅ 正确版本：`hardware/测试工具/DMTool v2.1.6.0-FD加速(1).exe`

**经验教训**：
- 达妙电机调试必须使用配套版本的 DMTool
- 文档要求 V2.0.0.0 及以上版本，但实际需要 v2.1.6.0-FD 才能稳定通信
- 遇到串口乱码、ID 错乱、读参数无响应等问题，**先排查工具版本**

### 6. 确立"复刻基线维护原则"

**背景**：检查上游时发现 MotorBridge 有 50 个新提交，引发对项目同步策略的根本讨论。

**核心结论**：
- 项目定位是**复刻集成工程**，不是单一软件包维护
- 目标：**持续追上游最新进展**，但 main 分支只接受**验证过的最新**
- 一句话原则：追最新，但不盲目追最新；更新必须记录来源、原因、验证结果和回滚点
- 三层结构：sync 分支（试升级）→ main（已验证的最新）→ baseline tag（阶段性快照）
- 所有 4 个核心依赖都应 fork 到 fanhao375 账号下（Seeed/MotorBridge/reBotArm_control_py/Star-Arm-102）
- 主仓库 submodule 永远只指向自己的 fork
- 每次同步检查都要记录，不同步也要写原因 + 下次触发条件
- Submodule 升级必须走 sync 分支 + 验证流程
- 不擅自 push、不擅自改 .gitmodules、不擅自升级 submodule

**落地动作**（阶段 A，今天完成）：
- 新建 `复刻基线维护原则.md`（主仓库根目录，作为最高规则文档）
- 更新 `项目总览.md` AI 接手提示（明确读文档顺序 + 核心定位）
- 本日志记录决策过程

**核心定位修正记录**：

- 用户指出表述不准确："不追最新"会误解成保守不动，实际是"追最新但有质量门"
