### 0.1 工具链架构（要先明白）🟢

```text
┌──────────────────────────────┐
│  浏览器（Chrome/Edge）        │  打开 https://rebot-devarm.w0x7ce.eu/
│  └ Web UI 按钮 + 拖滑块界面   │  ← 官方云端托管，不用装
└──────────────┬───────────────┘
               │ WebSocket  (127.0.0.1:9002)
               ▼
┌──────────────────────────────┐
│  motorbridge-gateway 进程     │  ← 本地装的，已就绪
│  作用：把 WS 指令翻译成 CAN 帧 │     Win: E:\Python312\Scripts\
│                              │     WSL: /usr/local/bin/
└──────────────┬───────────────┘
               │ USB 串口
               ▼
┌──────────────────────────────┐
│  USB-CAN 转接板（达妙官方）    │
└──────────────┬───────────────┘
               │ CAN 总线（CAN-H / CAN-L / GND）
               ▼
        7 个达妙电机（0x01..0x07）
```

**核心理解**：
- **Web UI 在云端、不用装**，浏览器打开 URL 即可
- 你电脑上跑的是 **gateway 进程**，负责"浏览器 ↔ USB-CAN ↔ 电机"的桥接
- 浏览器和 gateway 是**通过 localhost:9002 WebSocket** 通信
- 三条路殊途同归：Web UI 按钮 / `motorbridge-cli` 命令行 / Python `motorbridge` 包，**都是发同一套 JSON op 指令到 gateway**

## 3. 阶段 1：单电机烧 ID（×7）

### 3.1 电机 ID 分配（🟢 来自 arm.yaml + gripper.yaml）

| 关节 | motor_id | feedback_id | model | 说明 |
|---|---|---|---|---|
| J1（底座） | `0x01` | `0x11` | **4340P** | 大关节 |
| J2 | `0x02` | `0x12` | **4340P** | 大关节 |
| J3 | `0x03` | `0x13` | **4340P** | 大关节 |
| J4 | `0x04` | `0x14` | **4310** | 小关节 |
| J5 | `0x05` | `0x15` | **4310** | 小关节 |
| J6（腕） | `0x06` | `0x16` | **4310** | 小关节 |
| 夹爪 | `0x07` | `0x17` | **4310** | 小关节 |

规律：`feedback_id = 0x10 + motor_id`；大关节（DM-J4340P）×3，小关节（DM-J4310）×4。

电机出厂默认 `motor_id = 0x01`。**所以 J1 不需要烧（如果出厂值就是想要的）；其他 6 个必须烧**。

### 3.2 用 MotorBridge Web UI 烧 ID（推荐路径）

**为什么用 Web UI 而不是 DMTool**：见对话记录 — DMTool 版本碎片化（v1.6.8.8 ❌ / v2.1.6 ✅ / v1.8 ❓），Web UI 是上游主推一站式方案。

#### 步骤 1：硬件接线（单个电机）
```
24V 电源 → 信号电源隔离板 → 单个电机的 XT30 2+2 接口
USB-CAN 板 → 电机的 CAN 口
USB-CAN 板 → 电脑 USB
```

#### 步骤 2：查 USB-CAN 板的 COM 口
Windows：设备管理器 → 端口（COM 和 LPT）→ 找 USB-Serial / 达妙转接板，记下 `COM3` 还是 `COM4`。

#### 步骤 3：启动本机 gateway

```powershell
motorbridge-gateway --bind 127.0.0.1:9002 --vendor damiao --transport dm-serial --serial-port COM3 --serial-baud 921600
```
- 把 `COM3` 改成你实际的 COM 口
- 这个窗口**别关**，要一直跑
- 出错先看下面 §7 已知坑

#### 步骤 4：浏览器连入 Web UI

打开 https://rebot-devarm.w0x7ce.eu/，连接状态变绿色 = 跟你本机 gateway 通了。

#### 步骤 5：在 Web UI 烧 ID 🟢

Web UI 背后调的是 WebSocket JSON 协议（参考 `software/MotorBridge/integrations/ws_gateway/README.zh-CN.md`）。烧 ID 实际是 3 步：

**5.1 扫描总线（确认当前电机 ID）**

Web UI 点 "Scan" 按钮 = 发送
```json
{"op":"scan","start_id":1,"end_id":16,"feedback_base":16,"timeout_ms":100}
```

预期返回：单电机出厂默认 `motor_id=0x01`，扫到的是 1。

**5.2 写 ID（带验证 + 持久化）**

Web UI 改 ID 按钮 = 发送
```json
{
  "op":"set_id",
  "vendor":"damiao",
  "old_motor_id":1, "old_feedback_id":17,
  "new_motor_id":2, "new_feedback_id":18,
  "store":true,
  "verify":true
}
```

参数解释：
- `old_motor_id` / `old_feedback_id`：当前电机的 ID（出厂 1 / 17）
- `new_motor_id` / `new_feedback_id`：要写入的新 ID（按 §3.1 表填）
- `store: true`：**持久化到电机内部 flash**（断电不丢失），必须 true
- `verify: true`：写入后立刻读回校验

**5.3 单独验证（可选）**

```json
{"op":"verify","motor_id":2,"feedback_id":18,"timeout_ms":1000}
```

返回 `{"ok":true,...}` = 烧录成功。

**🟡 待你实操后补**：
- [ ] Web UI 上对应按钮叫什么（"Set ID" / "改 ID" / "写入"）
- [ ] 烧完是否需要断电再上电（理论上 `store:true` 直接生效不用断电，待验证）
- [ ] 实际界面截图（占位）

#### 步骤 6：烧完一个换下一个 ⚠️

**断电** → 拔 CAN 线 → 换下一个电机 → 接线 → 重启 gateway → 烧。

**绝对不能两个 motor_id = 0x01 的电机同时挂在 CAN 总线上**——会冲突无法识别。

### 3.3 单电机自测（验证烧录成功）

跑 `software/reBotArm_control_py/example/1_damiao_text.py`：

```python
# ⚠️ 跑前必须改 3 个配置
CHANNEL    = "COM3"     # Win: COM3 / WSL2: /dev/ttyACM0
MOTOR_ID   = 0x01       # 改成你刚烧的 ID
FEEDBACK_ID = 0x11      # = 0x10 + MOTOR_ID
MODEL      = "4310"     # ⚠️ 默认是 "4340P"，小关节必须改为 "4310"
```

## 4. 阶段 2：整机零点设定

### 4.1 装好整机 → 接好 CAN 总线

7 个电机的 CAN 全部挂上同一根 CAN 总线（USB-CAN 板）。

启动 gateway 时不指定 `--motor-id`，让它自动扫：

```powershell
motorbridge-gateway --bind 127.0.0.1:9002 --vendor damiao --transport dm-serial --serial-port COM3 --serial-baud 921600
```

Web UI 应该显示 7 个电机全部识别到（0x01..0x07）。

### 4.2 标零点协议层 🟢

**Web UI 一步记录零点 = 发送 `set_zero_position` op**（参考 `ws_gateway/README.zh-CN.md`）：

```json
{"op":"set_zero_position"}
```

**前提**：调用前必须先用 `set_target` 选中要标的电机（或 gateway 启动时已经指定 `--motor-id`）。

#### 标准做法（来自 `MotorBridge/docs/zh/operation_manual.md §9` 夹爪校准流程）

> ⚠️ 达妙电机用单圈编码器，**断电零点会丢失**。每次重新装机后都要重新标。

逐关节流程：

1. **MIT 模式低力矩"推"到机械限位**（碰到机械停止就停，不会硬撞）
   ```json
   {"op":"set_target","vendor":"damiao","motor_id":1,"feedback_id":17,"model":"4340P"}
   {"op":"enable"}
   {"op":"mit","pos":-5.0,"vel":0,"kp":5.0,"kd":0.5,"tau":0,"continuous":true}
   ```
   - `kp=5, kd=0.5, tau=0`：力矩极低，安全
   - 大关节 4340P 用上面参数；小关节 4310 同参数也可

2. **等反馈稳定**（`vel ≈ 0` 且 `pos` 不变 = 顶到位了）
   ```json
   {"op":"state_once"}
   ```

3. **一键写入零点**
   ```json
   {"op":"set_zero_position"}
   {"op":"store_parameters"}   // 持久化到 flash
   ```

4. **下一关节**：换 `set_target` 重复 1-3。

## 8. DMTool ↔ Web UI ↔ 命令行 三路对照表

视频是 4 月初拍的，大概率全程 DMTool。三种工具最终都是发同样的 CAN 帧，只是壳子不同：

| 任务 | DMTool 操作 | Web UI 操作 | 命令行 / JSON 协议（🟢 已确认） |
|---|---|---|---|
| 连接 | 选 COM 口 + 波特率 | 浏览器打开 URL，gateway 已启动 | `motorbridge-gateway --transport dm-serial --serial-port COM3 --serial-baud 921600` |
| 扫描电机 | "搜索"按钮 | "Scan" 按钮 | `{"op":"scan","start_id":1,"end_id":16,"feedback_base":16}` |
| 选中某个电机 | 列表里点一行 | 列表点一行 | `{"op":"set_target","vendor":"damiao","motor_id":1,"feedback_id":17,"model":"4340P"}` |
| 写 motor_id（烧 ID）| "修改 ID"对话框 | 🟡 待看 | `{"op":"set_id","old_motor_id":1,"old_feedback_id":17,"new_motor_id":2,"new_feedback_id":18,"store":true,"verify":true}` |
| 使能 / 失能 | "Enable" / "Disable" | 同名按钮 | `{"op":"enable"}` / `{"op":"disable"}` |
| 切 MIT 模式控制 | MIT 标签页输入 pos/kp/kd | MIT 滑块 | `{"op":"mit","pos":0,"vel":0,"kp":20,"kd":1,"tau":0,"continuous":true}` |
| 切位置-速度模式 | POS_VEL 标签页 | POS_VEL 滑块 | `{"op":"pos_vel","pos":3.1,"vlim":1.5,"continuous":true}` |
| **设零点** ⭐ | "保存当前位置为零点"按钮 | **"一键零点校准"按钮** | `{"op":"set_zero_position"}` |
| 持久化参数到 flash | "保存"按钮 | 🟡 待看 | `{"op":"store_parameters"}` |
| 读实时反馈 | "状态"窗口 | 实时 pos/vel/torque 显示 | `{"op":"state_once"}` / `{"op":"poll_feedback_once"}` |
| 单电机调试（拖动条）| 拖动 pos 输入 | "拖拽式电机控制" | 循环发 `mit` 或 `pos_vel` |
| 急停 | "Stop"按钮 | 同名按钮 | `{"op":"stop"}` |
| 清错误 | "Clear Error" | 同名 | `{"op":"clear_error"}` |

> 🟢 JSON 协议列来源：`software/MotorBridge/integrations/ws_gateway/README.zh-CN.md` 第 177 行起"入站命令示例"

> 🟡 实操时填："Web UI 操作"列里那几个 🟡 待看，等你今天 Web UI 一打开就能补
