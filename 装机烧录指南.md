## 3. 阶段 1：单电机烧 ID（×7）

### 3.1 电机 ID 分配（🟢 来自 arm.yaml + gripper.yaml）

| 关节 | motor_id | feedback_id | model | 说明 |
|---|---|---|---|---|
| J1（底座） | `0x01` | `0x11` | **4340P** | 大关节 |
| J2 | `0x02` | `0x12` | **4340P** | 大关节 |
| J3 | `0x03` | `0x13` | **4340P** | 大关节 |
| J4 | `0x04` | `0x14` | **4310** | 小关节 |
| J5 | `0x05` | `0x15` | **4310** | 小关节 |
| J6（腕） | `0x06` | `0x16` | **4310** | 小关节 |
| 夹爪 | `0x07` | `0x17` | **4310** | 小关节 |

规律：`feedback_id = 0x10 + motor_id`；大关节（DM-J4340P）×3，小关节（DM-J4310）×4。

电机出厂默认 `motor_id = 0x01`。**所以 J1 不需要烧（如果出厂值就是想要的）；其他 6 个必须烧**。

### 3.2 用 MotorBridge Web UI 烧 ID（推荐路径）

**为什么用 Web UI 而不是 DMTool**：见对话记录 — DMTool 版本碎片化（v1.6.8.8 ❌ / v2.1.6 ✅ / v1.8 ❓），Web UI 是上游主推一站式方案。

#### 步骤 1：硬件接线（单个电机）
```
24V 电源 → 信号电源隔离板 → 单个电机的 XT30 2+2 接口
USB-CAN 板 → 电机的 CAN 口
USB-CAN 板 → 电脑 USB
```

#### 步骤 2：查 USB-CAN 板的 COM 口
Windows：设备管理器 → 端口（COM 和 LPT）→ 找 USB-Serial / 达妙转接板，记下 `COM3` 还是 `COM4`。

#### 步骤 3：启动本机 gateway

```powershell
motorbridge-gateway --bind 127.0.0.1:9002 --vendor damiao --transport dm-serial --serial-port COM3 --serial-baud 921600
```
- 把 `COM3` 改成你实际的 COM 口
- 这个窗口**别关**，要一直跑
- 出错先看下面 §7 已知坑

#### 步骤 4：浏览器连入 Web UI

打开 https://rebot-devarm.w0x7ce.eu/，连接状态变绿色 = 跟你本机 gateway 通了。

#### 步骤 5：在 Web UI 烧 ID（🟡 TODO 实操后填）

- [ ] Web UI 里"写入 motor_id"的菜单/按钮在哪
- [ ] 是先输入 `motor_id` 再输入 `feedback_id`，还是只写 `motor_id` 由系统算 `feedback_id`
- [ ] 烧完是否需要断电再上电生效
- [ ] 怎么验证写入成功（读回 / ping）

#### 步骤 6：烧完一个换下一个 ⚠️

**断电** → 拔 CAN 线 → 换下一个电机 → 接线 → 重启 gateway → 烧。

**绝对不能两个 motor_id = 0x01 的电机同时挂在 CAN 总线上**——会冲突无法识别。

### 3.3 单电机自测（验证烧录成功）

跑 `software/reBotArm_control_py/example/1_damiao_text.py`：

```python
# ⚠️ 跑前必须改 3 个配置
CHANNEL    = "COM3"     # Win: COM3 / WSL2: /dev/ttyACM0
MOTOR_ID   = 0x01       # 改成你刚烧的 ID
FEEDBACK_ID = 0x11      # = 0x10 + MOTOR_ID
MODEL      = "4310"     # ⚠️ 默认是 "4340P"，小关节必须改为 "4310"
```

跑起来后试 `enable` → `mit` 命令，看电机能不能动。

## 4. 阶段 2：整机零点设定

7 个电机的 CAN 全部挂上同一根 CAN 总线（USB-CAN 板）。

motorbridge-gateway --bind 127.0.0.1:9002 --vendor damiao --transport dm-serial --serial-port COM3 --serial-baud 921600

| 现象 | 原因 | 解法 |
|---|---|---|
| 串口乱码 / ID 错乱 / 读参数无响应 | DMTool 用了 v1.6.8.8 旧版 | 用 Web UI 或 DMTool v2.1.6.0-FD |

| `pip install -e ...` 装的 wheel 没有新符号 | editable 模式不带预编译 .dll | 直接 `pip install --upgrade motorbridge` 拉 PyPI 预编译 wheel（5-09 验证） |
